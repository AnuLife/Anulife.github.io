浏览器关键渲染路径浅解#51

Anuluca     Date : 2021-03-03   Tags : 1

关键渲染路径指当浏览器接收到从服务器发送过来的HTML页面信息，在将其绘画渲染到屏幕上之前所做对一些操作，简称CRP。
通过了解这方面对知识可以更有利于我们对页面进行性能优化

§ 步骤

1. 生成DOM

浏览器从远程下载 Byte => 根据相应的编码 (如 utf8) 转化为字符串 => 通过 AST 解析为 Token => 生成 Node => 生成 DOM。

2. 生成CSSOM

浏览器解析css文件，生成CSSOM。

3. Render Tree

DOM和CSS·OM会一起生成Render Tree，只包含需要渲染的节点。

4. Layout

计算每一个元素在窗口内的确切位置和大小

5. Paint

栅格化所有元素，将元素转换为实际像素

§ 重排和重绘

  • 重排(Reflow)：元素的 位置发生变动 时发生重排，也叫回流。此时在 Layout 阶段，计算每一个元素在设备视口内的确切位置和大小。当一个元素位置发生变化时，其父元素及其后边的元素位置都可能发生变化，代价极高
  • 重绘(Repaint): 元素的 样式发生变动，但是位置没有改变。此时在关键渲染路径中的 Paint 阶段，将渲染树中的每个节点转换成屏幕上的实际像素，这一步通常称为绘制或栅格化

重排必定会造成重绘。
如果需要避免过多的重排重绘，需要使用 DocumentFragment 进行 DOM 操作，不过现在原生操作很少也基本上用不到

§ 白屏

这里白屏指设备从输入完网页链接后点击跳转开始到整个页面渲染完成，这个过程网页会呈现的白屏效果，减少白屏的时间就需要开发者优化网页第一次加载的性能。

§ 性能优化

  1. 减少DOM树渲染时间（譬如降低HTML层级、标签尽量语义化等等）
  2. 减少CSSOM树渲染时间（降低选择器层级等等）
  3. 减少HTTP请求次数及请求大小
  4. 将css放在页面开始位置
  5. 将js放在页面底部位置，并尽可能用defer或者async避免阻塞的js加载，确保DOM树生成完才会去加载JS
  6. 用link替代@import
  7. 如果页面css较少，尽量使用内嵌式
  8. 为了减少白屏时间，页面加载时先快速生成一个DOM树
