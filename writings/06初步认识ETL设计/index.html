初步认识ETL设计

Anuluca     Date : 2020-05-19   Tags : 1

本文是我还未正式处理过ETL需求前的个人理解，不具有任何参考性

什么是“ETL设计”

ETL是将业务的数据经过抽取、清洗转换之后加载到数据仓库的过程，目的是将企业中的分散、零乱、标准不统一的数据整合到一起，为企业的决策提供分析依据。
ETL的设计分三部分：数据抽取、数据的清洗转换、数据的加载。

数据抽取

这一步要明确几个问题：

  • 数据来源是哪几个业务系统？
  • 这些业务系统用的是哪些DBMS？
    如果是与DW同一个DBMS，则直接select就可以；如果不是，须通过ODBC的方式建立数据库链接，或者将元数据表导出，再导入至ODS中，还有一种就是开发数据接口
  • 确认是否存在手工数据？如果存在，量有多大？
  • 是否存在非结构化(?)数据？
    依具体情况可能还有别的问题，列举出所有问题后才能开始数据抽取。

举个例子：为了系统查询需要的销售数据，从数据源抽取数据

数据的清洗转换

ODS-DW的过程

  • ODS清洗：过滤错误数据、不符合要求的数据、不完整的数据、重复数据，过滤掉的数据可以备份到excel表里或者新数据表里；
  • 数据转换：举个例子，某个烟品在国家局系统里编号是6000XXXX，但在营销系统里格式是36XXXXX，这个转换的过程就是数据转换；
  • 数据粒度的转换：举个例子：原表每个物流单都有各自的运费，但我只需要每个车次的总运费，粒度由小变大，这一步就是粒度转换；
  • 加工数据：举个例子：我从元数据获得了各地市里程和年费率的数据，但我只需要运费数据，则在数据清洗转换的过程中将元数据进行计算，这一步就是加工数据

日志&报警

  • 日志记录：
    1.执行过程日志（sql之类的）
    2.报错日志
    3.总体时间、XML信息、是否成功（自动生成的日志）
  • 报警：
    通常在报错日志之后，JAVA里自动给管理员发送邮件（数据更新时间、报错数据生成的报表），也有的系统会在发送成功的时候也调用邮件方法。

ETL设计的实现方法

网上找到的实现方法一种是借助ETL工具（如Oracle的OWB、SQL Server 2000的DTS、SQL Server2005的SSIS服务、Informatic等）实现；另一种是用写SQL实现，优缺点就是灵活性、编写技术难度方面之类的。

一个简单BI项目环节的梳理：https://wenku.baidu.com/view/2ad8143ba26925c52cc5bfab.html

备注

ETL设计在网上的相关介绍文档基本是同一份copy，不知道谁才是原作者，所以这篇文章只结合目前手头的项目做出我对ETL设计的个人理解。
